Open Source workflow - ML

I am writing this to introduce you guys with marklogic workflow process.

  • I am new to team what should I do to get the marklogic setup done?

    First of all welcome to the team, Marklogic has private github repo which is generally shared with globant team working for marklogic. I assume you must be having github account with you. (If not do so please register yourself here). Once you have your account with you share your username with your project / team manager to get you assigned in marklogic organization.

  • I can see marklogic organization in my github account. What should be the next step to setup this project on local system.

    • First of all you need to fork this porject (marklogic/ opsdirector) in your github account.
    • Let's say your username is abc. Once you are done with forking you will see opsdirector in your local account like abc/opsdirector.
    • Now start cloning this abc/opsdirector on your local system.
    • Once you are done with cloning , go with readme.md of abc/marklogic.

  • Opsdirector setup is done on my local system, what should steps I should follow before start of actual development.

    • As we are following open source workflow, it means each team member will have opsdirector in their account and then they will start coding into their branch.
    • It might required that you need to keep yourself in sync with other developer.
    • To achieve the same, I would like you to run few commands.
    • run git remote -v : this command shows how many remote added.
    • run git remote add remote_name remote_url : this commnd will add remote with this remote_name.
    • run again git remote -v.

  • I have setup the marklogic along with the remote of each member in my team. What to do next.?

    • Before start the work for day, we need to run some set of cammand to get the latest changes from remotes.
    • run git fetch --all. we always make sure that our branch which we are working upon must be rebased on to marklogic develop.

  • How would I pull the latest changes of marklogic/develop or any other remote branch into in my current branch.

    • I assume you have run git fetch --all to fetch the changes from all remote.
      • We geneally prefer rebasing the changes whenever we pull the latest changes of any remote into your branch.
      • Lets assume you are working on branch feature/abc which is derived from develope branch and you want to merge marklogic/develop changes into this branch. So you can rebase it by running some cammand.
      • git rebase marklogic/develop.
      • Once you run this command you will be prompted with further option on the terminal or screen.
      • Once you are done with rebase you can check your commits on the top of marklogic/develop. run this command git log

  • I have completed with my task and I want it push this changes to marklogic/develop. What steps are required for the same?

    • As you know, we are following open source workflow so we will be creating a pull request to merge our changes with marklogic/develop.
    • git status
    • git add filename or git add . if there are some untracked changes needs to be committed.
    • git commit -m 'message'.
    • git rebase -i marklogic/develop. Here you will be asked to pick and squash some commit. We genereally keeps the first commint and squash rest commit to make it one commit.
    • Read this for better understanding.
    • once you are done with this, create a pull request on github with marklogic/develop.

  • Now my PR is merged into marklogic/develop, what step should I follow ?

    Once your PR is in marklogic/develop, delete your branch and pull the latest marklogic/develop into your develop branch.

  • Quick Commands at glance:-

    • git status
    • git add file-name or git add . if any unstage changes.
    • git commit -m 'message'
    • git fetch --all If remote has changes then follow below step.
    • git rebase -i marklogic/develop.
    • squash commit and make it single.
    • git log to check the history.
    • git push remote-name branch-name.
